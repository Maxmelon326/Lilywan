This project simulates how a FinTech company builds a layered credit risk engine to evaluate new and existing users before making lending decisions. Two distinct risk models were developed:
New User Model: Evaluates applicants using only self-declared demographic and financial information (e.g., age, occupation, income), simulating users with no prior credit history.
Existing User Model: Incorporates behavioral and historical credit data (e.g., credit score, limit utilization, past defaults) to provide a more informed risk assessment.
Using a real-world structured dataset, two models were trained and compared, simulated approval decisions, and an interactive demo was deployed to replicate a simplified credit review system.
Key Results
A credit decision engine was simulated using risk thresholds to categorize applicants into Auto-Approved, Manual Review, or Rejected groups.
A Streamlit demo app was built to replicate how credit analysts or automated systems assess applicants in real-time.
Tools Used
Python (Pandas, LightGBM, SHAP, imbalanced-learn), Google Colab, Streamlit, Matplotlib, Seaborn, SMOTE.
Outcome
This project demonstrates how FinTech companies approach risk modeling with different data layers, highlights the gap between new and existing user assessments, and showcases how even modest models can inform real-world lending strategies when paired with business logic. The deployed demo provides a hands-on view of how machine learning integrates into credit decisioning systems.

This project leverages Azure Synapse Analytics and Power BI to analyze content features and publishing trends of Netflix movies and TV shows. The dataset includes metadata such as genres, languages, countries, durations, ratings, popularity, and release years. Using PySpark in Azure Synapse Notebooks, the data was cleaned and unified before being stored in a Dedicated SQL Pool. An automated Data Pipeline was developed to streamline data integration, and the results were visualized through interactive Power BI dashboards for further analysis.
Key Analyses
1. Structural analysis of genres, countries, languages, durations, and ratings by platform (TV vs Movie).
2. Trends in release year to understand Netflix's publishing strategy over time.
3. Global content strategy via language and regional content distribution.
4. Seasonal or monthly release patterns to detect holiday-focused content strategies.
Tools Used
Azure Synapse Analytics, PySpark, Azure Data Factory, Power BI (Desktop & Mobile).
Outcome
An end-to-end cloud-based data analytics workflow was established, showcasing the use of Microsoft tools for large-scale data engineering. The final result is a Power BI dashboard that reveals Netflix's content evolution and platform behavior insights.

This project simulates an A/B test to evaluate two different strategies for monetizing emoji reactions in a chat app.
Group A uses a simplified flow, sending all users directly into the payment funnel.
Group B introduces pre-qualification steps to filter low-intent users before showing payment options.
Using a simulated dataset, I analyzed user behavior through five funnel steps and performed statistical comparisons between the two groups.
Key Results
1. Group A achieved significantly higher conversion rates at key monetization steps (payment click & success).
2. Statistical testing (t-tests) confirmed meaningful differences between the groups.
3. Funnel visualizations and bar plots provided clear insights into user drop-off and conversion performance.
Tools Used
Python (Pandas, Matplotlib, SciPy), Google Colab, A/B testing principles, T-tests.
Outcome
Group A's simplified design led to better performance, highlighting how reducing friction in conversion funnels can improve monetization.
